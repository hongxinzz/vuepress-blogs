# 原型链

# 如果创建对象

  • 在Javascript中我们可以通过三种方式创建对象。
    var obj1 = {};
    var obj2 = new Object();
    var obj3 = Object.create({});

# 通过面向对象分析原型链

function Test(){}

var test = new Test();

  • 在上述代码中我们创建了一个Test函数，并通过new关键字将实例赋值给了test。

  • 接下来我们可以尝试在控制台 输出Test和 test。

Test

  • 构造函数中有2个属性分别为__proto__和prototype。
  • 其中 prototype还有一个constructor指向Test本身。

test

  • 而在实例化对象test 我们只看到了__proto__。

# 小总结

  • prototype 是函数中独有的，并且function本质是一个对象所以也有__proto__。

# 原型链的关系

# 从实例化对象 test 入手

  • 上图中我们发现test.__proto__也有一个constructor并且也指向了Test,所以我们可以得出。
test.__proto__ === Test.prototype // true
  • test.__proto__ 中还有一个__proto__属性，我们可以推断出test.__proto__.__proto__的constructor是不是也是指向一个构造函数本身呢？我们可以在控制台输出来看下。
test.__proto__.__proto__ === Object.prototype // true
  • 我们可以继续向上推 test.__proto__.__proto__.__proto__，结果输出了null,我们可以推断出，原型链的顶端就是null。
test.__proto__.__proto__.__proto__ // null

  • 到此我们分析完了test上的原型链。

# 从构造函数 Test 入手

prototype

  • 上图中我们发现Test.prototype.__proto__也有一个constructor并且也指向了Object,所以我们可以得出。这个和分析test是一样的。
Test.prototype.__proto__ === Object.prototype // true

Test.prototype.__proto__.__proto__ //null

__proto__

注意: 我们刚刚说过构造函数Test本质也是一个对象。

  • 上图中我们发现Test.__proto__也有一个constructor并且指向了Function,我们就知道了原来function是通过Function产生的。
Test.__proto__ === Function.prototype // true
  • 图中的 Test.__proto__.__proto__又指向了向了Object，所以我们也可以知道.
Test.__proto__.__proto__  === Object.prototype // true
Function.prototype.__proto__ === Object.prototype //true
  • 到此我们分析完了Test上的原型链。

在上面的分析中我们清楚的知道

  • 实例化对象和我们的Object有直接关联，我们可以称为Object是数理化对象的宿主。
  • 而在函数对象上的 __propt__ 会指向一个 Function构造函数，而Function构造函数的上一层就是 Object这个构造函数。

宿主关系：实例化对象 -> Function -> Object

# instanceof

为了验证我们刚刚的总结是否真确，我们可以通过instanceof用来判断运算符用于检测构造函数的prototype属性是否出现在某个实例对象的原型链上。

//实例和构造函数Tets的关系
test instanceof Test //true
//构造函数
Test instanceof Function //true
Test instanceof Object //true
//实例化对象
test instanceof Object //true
test instanceof Function //false

// Function构造函数
Function instanceof Object //true
